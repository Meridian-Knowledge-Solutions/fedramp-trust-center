Secure Configuration

Meridian LMS — Secure Configuration Baseline

This baseline documents the hardened configuration enforced across the Meridian LMS production boundary, validated continuously by the FedRAMP 20x KSI pipeline. Settings below are drawn from infrastructure-as-code and verified against live cloud state.

All values are enforced via Terraform and re-validated every pipeline run; drift triggers an automatic Significant Change evaluation.

Identity & Access

  • Phishing-resistant MFA required org-wide (PIV/CAC + WebAuthn).
  • SSO via SAML 2.0; session timeout 15m idle, 12h absolute.
  • Least-privilege IAM permission sets, reviewed quarterly.

Logging & Monitoring

CloudTrail policy excerpt

# audit logging — all regions, log-file validation on
resource "aws_cloudtrail" "audit" {
  is_multi_region_trail        = true
  enable_log_file_validation   = true
  kms_key_id                   = aws_kms_key.audit.arn
}

Logs stream to a centralized, immutable account with 1-year retention and anomaly detection via GuardDuty + Security Hub. See the Transparency Console for the live evidence stream.
